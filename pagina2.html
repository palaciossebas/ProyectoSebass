Diseño Web - Trimestre 1

¿Qué es un hosting?

El hosting (alojamiento web) es un servicio que permite almacenar y publicar sitios web en internet. Funciona como el espacio donde se guardan los archivos, imágenes, videos, bases de datos y otros elementos necesarios para que una página funcione y esté disponible las 24 horas del día.

Tipos de hosting

Hosting compartido: Varios sitios web comparten un mismo servidor. Es económico y fácil de usar, ideal para páginas pequeñas o personales.

VPS (Servidor Privado Virtual): Ofrece más recursos y control que el compartido, ya que el servidor se divide virtualmente para cada usuario.

Hosting dedicado: Todo el servidor es para un solo cliente. Tiene alto rendimiento, pero es más costoso y requiere conocimientos técnicos.

Hosting en la nube (cloud hosting): Utiliza varios servidores interconectados, ofreciendo alta disponibilidad, escalabilidad y seguridad.

Hosting gratuito: No tiene costo, pero suele tener muchas limitaciones y publicidad. Sirve para pruebas o proyectos muy simples.

Características importantes

Espacio en disco: Capacidad para guardar archivos.
Ancho de banda: Cantidad de datos que puede transferir.
Soporte técnico: Ayuda del proveedor en caso de fallos.
Seguridad: Protección contra ataques y pérdidas de datos.
Panel de control: Herramienta para gestionar el sitio (como cPanel).

Ejemplos de proveedores de hosting

Hostinger, GoDaddy, Bluehost, SiteGround, Google Cloud, AWS, Microsoft Azure.

Importancia del hosting

Un buen servicio de hosting garantiza que una página web funcione de forma rápida, segura y estable, lo cual es clave para la experiencia del usuario y el posicionamiento en buscadores (SEO).

Segunda clase: GitHub

GitHub ¿Qué es GitHub?
GitHub es una plataforma en línea que permite almacenar, gestionar y colaborar en proyectos de programación usando Git, un sistema de control de versiones.

Fue creada en 2008 y en 2018 fue adquirida por Microsoft. Hoy en día es una de las herramientas más usadas por desarrolladores, empresas y estudiantes para trabajar en proyectos de software de manera colaborativa.

¿Qué es Git?
Git es un sistema de control de versiones desarrollado por Linus Torvalds (el creador de Linux). Permite llevar un registro de todos los cambios que se hacen en los archivos de un proyecto, facilitando el trabajo en equipo y la recuperación de versiones anteriores.

¿Para qué sirve GitHub?
Guardar código en la nube.
Controlar versiones del proyecto.
Colaborar con otros programadores.
Revisar, comentar y mejorar el código.
Compartir proyectos públicamente o mantenerlos privados.

Principales funciones de GitHub
Repositorios (repositories): Son como carpetas donde se guarda todo el código y documentación del proyecto.
Commits: Son registros de cambios guardados en el repositorio.
Branches (ramas): Permiten trabajar en versiones paralelas del código sin afectar la principal.
Pull requests: Sirven para proponer cambios y discutirlos antes de integrarlos.
Issues: Herramientas para reportar errores o sugerir mejoras.
GitHub Pages: Permite publicar páginas web directamente desde un repositorio.

Ventajas de GitHub
Trabajo colaborativo desde cualquier parte del mundo.
Historial completo de cambios.
Facilita el aprendizaje del trabajo en equipo en desarrollo de software.
Integración con herramientas como VS Code, Slack, Trello, etc.
Uso gratuito para proyectos públicos y privados limitados.

Ejemplos de uso
Equipos de desarrollo creando aplicaciones web, móviles o de escritorio.
Estudiantes subiendo proyectos escolares.
Empresas gestionando código privado con equipos grandes.
Publicación de portafolios profesionales en GitHub Pages.

Importancia de GitHub
GitHub es hoy una herramienta esencial en la industria del software, no solo para almacenar código, sino para gestionar todo el proceso de desarrollo de un proyecto. También es muy útil para construir un perfil profesional como programador, ya que permite mostrar habilidades y experiencia públicamente.

Esta es una plataforma colaborativa usada ampliamente en el desarrollo web y de software.
